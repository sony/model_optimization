Get Hardware Model¶

model_compression_toolkit.get_model(fw_name, hw_name)¶

Get a FrameworkHardwareModel by the hardware model name and the framework name. For now, it supports frameworks ‘tensorflow’ and ‘pytorch’. For both of them the hardware model can be ‘default’,’tflite’, or ‘qnnpack’.

Parameters

  • fw_name (str) – Framework name of the FrameworkHardwareModel.

  • hw_name (str) – Hardware model name the model will use for inference.

Returns

A FrameworkHardwareModel object that models the hardware and attaches a framework information to it.

Return type

FrameworkHardwareModel

Note

For now, the only information from HardwareModel that MCT uses are the values of QuantizationMethod (for weights and activations) from the default QuantizationConfigOptions that is set to the HardwareModel.

  • MCT will use more information from HardwareModel gradually, in the future.
